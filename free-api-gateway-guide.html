What is it

How an API Gateway works

An API gateway is a server that sits between your clients and your backend services. Every request passes through it — so you get one place to handle auth, rate limiting, logging, routing, and more, instead of coding those things into every service.

The gateway handles cross-cutting concerns once, so each backend service stays lean and focused on its own business logic.

🔐
Authentication & Authorization
Validates API keys, JWT tokens, OAuth2. No auth code in every microservice.
⏱️
Rate Limiting
Limit requests per client/IP per second or day. Protects your backend from overload.
↔️
Routing & Load Balancing
Route requests to the right service. Canary deploy, A/B test, weighted traffic splits.
📊
Observability
Centralized logs, metrics, tracing. See every request across all your services in one place.
🔄
Request Transformation
Modify headers, rewrite paths, aggregate multiple backend calls into one response.
💾
Caching
Cache responses at the gateway level. Reduce backend load for repeated identical requests.

7 Best Free API Gateways

01
Kong Gateway (OSS)
The most widely adopted open-source gateway
High perf Kubernetes Plugin-rich
▾

Kong is built on NGINX and LuaJIT, making it fast and stable. It has the largest open-source community and plugin ecosystem — 60+ bundled plugins for auth, rate limiting, logging, transformations, and more. Kong Konnect (the commercial tier) adds managed cloud, but the OSS version is fully functional for most production use cases.

Strengths
  • +Largest plugin ecosystem (~60 OSS, hundreds community)
  • +Massive community — 300K+ monthly instances, 15M+ Docker pulls
  • +Kubernetes-native via Kong Ingress Controller
  • +REST Admin API + web UI (KongHQ)
  • +Strong enterprise upgrade path (Kong Konnect)
  • +AI proxy plugins (LLM support) added in 2024–25
Weaknesses
  • –Requires PostgreSQL or Cassandra for HA config store
  • –1–3ms latency overhead with plugins (vs <1ms APISIX)
  • –Advanced features (RBAC, analytics) locked to Enterprise
  • –Primary plugin language is Lua — less familiar to most devs
Specs
LicenseApache 2.0
LanguageLua / NGINX
Max throughput~30K RPS (single node)
Latency add1–3ms
Config storePostgreSQL / Cassandra
ProtocolsHTTP, HTTPS, gRPC, WebSocket, TCP
Best for

Teams that want a battle-tested gateway with a massive plugin library and a clear path to enterprise. If your team knows Lua or needs a huge community, Kong is the default choice.

02
Apache APISIX
Highest throughput, fully dynamic, no DB on data path
Best perf Cloud-native Apache
▾

APISIX is built on NGINX + LuaJIT with etcd as the config store — meaning configuration is distributed and highly available without a database on the request path. It consistently benchmarks 2–4× faster than Kong with 30–50% lower latency. Used by Tencent, NASA, and others processing over 1 trillion API calls daily.

Strengths
  • +Best-in-class performance: sub-millisecond overhead, 23K+ QPS per node
  • +Fully dynamic — config changes applied with zero restarts
  • +100+ plugins; supports Lua, Go, Python, Node.js, Java plugins
  • +etcd backend — no single point of failure, no DB on data path
  • +Full Apache 2.0 license — no open-core gotchas
  • +Kubernetes Gateway API support, service mesh via GAMMA
Weaknesses
  • –Smaller community outside Asia-Pacific
  • –Documentation can be inconsistent in quality
  • –Dashboard/UI is less polished than Kong's
  • –Requires etcd knowledge for production deployment
Specs

Max throughput23K–50K+ QPS (single node)

Teams that need maximum throughput and lowest latency. Ideal for high-traffic microservices, cloud-native Kubernetes setups, and organizations who want a truly vendor-neutral open-source gateway.

03
Tyk
Full management stack in the OSS edition — analytics, portal, dashboard
Full APIM Dev portal Go-based
▾

Tyk is written in Go and is one of the few open-source gateways that gives you a full API management suite for free — including a developer portal, built-in analytics, and a dashboard. REST, GraphQL, TCP, and gRPC are all supported. It also supports multi-tenancy in the OSS version, which is rare.

Strengths
  • +Full APIM stack for free: gateway + analytics + dashboard + developer portal
  • +Multi-tenancy support in OSS (rare among free gateways)
  • +Written in Go — easy to extend, predictable memory usage
  • +Excellent GraphQL native support
  • +REST API and web dashboard for admin tasks
  • +Rated 4.8/5 on Gartner Peer Insights
Weaknesses
  • –Smaller plugin ecosystem than Kong
  • –Higher latency than APISIX/KrakenD (Go vs NGINX)
  • –UI has a learning curve for new users
  • –Community smaller than Kong's

04
KrakenD
Stateless, ultra-fast, built for BFF and API aggregation
70K RPS BFF pattern Stateless
▾

KrakenD is stateless — no database, no coordination between nodes, no shared state. It's purely declarative via a JSON config file. Its killer feature is backend aggregation: it can fan out one client request to multiple backends and merge the responses, implementing the Backend for Frontend (BFF) pattern elegantly. Backed by the Linux Foundation.

Strengths
  • +Highest raw throughput: 70K+ RPS on a single instance
  • +Truly stateless — no database, no Redis, no coordination
  • +Backend aggregation / BFF pattern built-in
  • +Declarative config via JSON — CI/CD friendly
  • +KrakenDesigner — visual config builder
  • +OWASP-compliant, data-agnostic (no data stored)
Weaknesses
  • –No built-in developer portal or analytics dashboard
  • –Less plugin flexibility than Kong or APISIX
  • –Smaller community
  • –EE version required for some advanced features
Specs

ProtocolsHTTP, HTTPS, gRPC, WebSocket
Best for

Teams building BFF layers, needing maximum throughput, or wanting a dead-simple deployment with no database dependencies. Perfect for pure performance and CI/CD-friendly declarative config.

05
Gravitee.io
Event-native: REST, WebSocket, async, and event-driven APIs
Full APIM EU-sovereign Async APIs
▾

Gravitee is an event-native API management platform — uniquely strong at both synchronous (REST) and asynchronous (WebSocket, Kafka, MQTT, SSE) API management. Founded in France, it's popular with European organizations that need data sovereignty and GDPR compliance. Includes a full APIM suite: gateway, developer portal, analytics, and access management.

Strengths
  • +Best async API support: WebSocket, Kafka, MQTT, SSE, SSE
  • +Full APIM suite: gateway + portal + analytics + access mgmt
  • +European-founded, strong GDPR and data sovereignty focus
  • +Policy-driven configuration — no custom code needed
  • +Community edition is fully featured for most use cases
Weaknesses
  • –More complex setup than lighter alternatives
  • –Lower throughput than APISIX or KrakenD
  • –Smaller community than Kong
  • –Java-based — higher memory footprint

06
Envoy Gateway
CNCF-standard, Kubernetes-native, service mesh backbone
K8s-native Service mesh L4/L7
▾

Envoy is a high-performance proxy originally built by Lyft, now a CNCF project. Envoy Gateway (launched 2022, now stable) implements the Kubernetes Gateway API spec using Envoy as the data plane. It's the reference implementation used in the CKA/CKAD certification exams. Best when you need a service mesh alongside your gateway (Istio, etc.) since Envoy powers both.

Strengths
  • +CNCF reference implementation for Kubernetes Gateway API
  • +Exceptional observability: native tracing, metrics, access logs
  • +Supports L4 and L7, HTTP/2, gRPC, WebSocket
  • +Powers Istio — unified gateway + mesh data plane
  • +Wasm filter extensibility for custom logic
  • +mTLS between services out of the box
Weaknesses
  • –Steep learning curve: xDS API, Wasm filters, complex config
  • –No built-in developer portal, API key management, or monetization
  • –Not a standalone API management solution — focused on infra/traffic
  • –Usually deployed alongside another gateway for API management

07
Traefik
Auto-discovery, Docker-native, zero-config for simple setups
Easy setup Docker/K8s Go
▾

Traefik is a cloud-native reverse proxy and load balancer with automatic service discovery. It reads Docker labels and Kubernetes annotations to configure itself — zero manual routing rules for simple setups. It's the easiest gateway to get started with and is extremely popular in Docker Compose environments. Written in Go and very resource-efficient.

Strengths
  • +Automatic service discovery — no manual route config needed
  • +Zero-config for Docker / Docker Compose setups
  • +Built-in Let's Encrypt / HTTPS termination
  • +Lightweight Go binary — minimal resource usage
  • +Good Kubernetes Ingress + Gateway API support
  • +Nice built-in dashboard for traffic overview
Weaknesses
  • –Enterprise features (distributed tracing, advanced auth) require paid Traefik Enterprise (~$600/mo)
  • –Less plugin flexibility than Kong or APISIX for complex API management
  • –Not suited for full API lifecycle management out of the box
  • –No developer portal or analytics built-in

Security deep-dive

Authentication Comparison

How each gateway handles identity verification — the most critical security layer.

🔑
API Key
Simple shared secret. Fast, stateless. Best for server-to-server over TLS. Not safe for browser/mobile clients.
🪙
JWT
Signed tokens with embedded claims. Self-contained — gateway validates without hitting a DB. Dominant for high-throughput APIs.
🔐
OAuth 2.0 / OIDC
Delegated authorization + identity. Standard for third-party access and SSO. Supports Okta, Auth0, Azure AD, Google.
🤝
mTLS
Mutual TLS — both client and server verify certificates. Zero-trust foundation for machine-to-machine. No application-layer tokens.
✍️
HMAC Signing
Request signature verification. Ensures integrity and authenticity. Used by AWS Signature, Tyk policy enforcement.
🏢
LDAP / Basic
Enterprise directory auth (Active Directory). Legacy but still common in intranet or on-prem environments.
